What I Built

A full internal ops platform for OOH advertising

This is a production-quality internal web application with ten fully functional modules: Dashboard, Inventory, Campaigns, Clients, Bookings, Analytics, Weather, Ops Queue, Users, and Reports. Every module is wired to a real Node.js REST API hosted on my own infrastructure, seeded with OOH data (31 inventory units, 8 campaigns, 5 clients, 101 bookings). Click any link, filter any table, open any campaign, and the data is real.

Core Features
📊

KPI Dashboard

Live revenue, impressions, campaign count, and utilization tiles. Built-in Rate Calculator for instant quotes. Live-refresh indicator with one-click data reload. Clickable alert notifications that route directly to the relevant module.

🗺️

Inventory Map

31 units across billboard, truckside, and DOOH types. Toggle between sortable table and Leaflet map. Rich map popups show rate, weekly impressions, and traffic score per unit. Click any unit to load its live weather risk panel.

📋

Campaigns

Full campaign management with status indicators, client assignments, date ranges, and budget tracking. Detail panel includes a per-unit revenue breakdown bar chart and full booking history.

📅

Bookings Timeline

Gantt-style visualization of 101 bookings across all inventory units. Bulk approve/reject for pending slots (checkbox multi-select, confirm dialog). Conflict detection enforced at the API level on every booking request.

🌦️

Weather / Install Risk

Live Open-Meteo forecast per billboard location. Surfaces rain, high-wind, and temperature flags relevant to vinyl installs and crew scheduling.

📈

Analytics

Revenue over time, impressions by channel, market share, and weekly reach by unit type (donut chart with cost-efficiency scoring per $1k spend). All derived from live data with no hardcoded values.

🏢

Clients

Client directory with per-client revenue chart, campaign utilization panel, and notes. Links to active campaigns and full booking history per advertiser.

🔧

Ops Queue

Install assignment tracking and field crew task queue. Surfaces pending installs, unit assignments, and status transitions for the Ops role, separate from Sales and Admin workflows.

👥

Users, Reports & UX

Admin user management with role assignment. Exportable reports view. Dark/light mode toggle with localStorage persistence. Clickable nav notifications per role (weather alerts, pending approvals, expiring campaigns).

Role-Based Access Control

Four roles, gated at API and UI

Protected routes and nav items filter per role. The API validates the JWT role on every mutation. Try each demo account to see the difference.
