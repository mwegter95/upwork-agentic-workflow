How I would scope the real build

Five pages, fast, mobile-first, accessible.

  • Site map. Home, About and Our Approach, Services, Team, Insurance, Contact and Intake. Five to seven pages depending on what you need.
  • Performance and a11y. Static hosting, sub-second loads, WCAG AA contrast, 44 by 44 touch targets, keyboard navigation, reduced-motion respected.
  • Trust depth. Named BCBA bios with credentials, plain-language services, insurance row, FAQ, parent-first tone throughout.
  • HIPAA-aware intake. Marketing form collects only name, email, phone, and a service interest dropdown, with a visible note about not entering medical details. Real intake routes through a HIPAA-eligible channel.
  • Editability. Optional headless CMS (Sanity or Contentful) if you want to update copy yourselves. Otherwise a clean static build keeps cost and complexity low.
  • SEO basics. Local schema markup, sensible titles and meta, sitemap, fast Core Web Vitals.

Kickoff questions

Three things I would ask before we start.

  • Page count and structure. Single scrolling page or a 5 to 7 page site? My recommendation is the latter; the demo shows the single-page version for speed.
  • CMS need. Do you want to edit copy yourselves after launch? If yes, I would add a small headless CMS layer. If not, a static build is cheaper to maintain.
  • Brand assets. Do you have a logo, color direction, or photography? If not, I can propose a small visual direction up front so we are aligned before I build.
