# Cover Letter — KS Global Estates

Your next visitor decides whether KS Global Estates is worth their time in about three seconds. That first impression needs to be undeniable.

I went ahead and built a working demo for you: **https://michaelwegter.com/work-samples/ks-global-estates**

It is a complete property discovery platform with a synchronized split-pane layout: a live Leaflet world map on one side and a filterable listing grid on the other. The mock database has 25 global properties across Dubai, London, New York, Paris, Singapore, Sydney, and more. Faceted filters (type, price, beds, baths, status, and region) update both views at once. Clicking a listing flies the map to that property and turns its price pin gold. Each property opens a detail panel with an image gallery, full specs, and an inquiry form. A favorites system persists across sessions. Every image is an actual house, condo, estate, or interior (no unrelated stock photography).

For the real site, I would extend this with a production backend (Node or Python), a CMS for managing listings, user authentication, and any features on your roadmap. Timeline and scope are worth a short conversation once you have seen the demo.

The design was built deliberately to avoid the common tells of AI-generated web work: no centered hero with an icon row beneath it, no uniform rounded-card grids, no Inter-only typography, no gradient blobs. The result uses Cormorant Garamond for display headings and DM Sans for the UI, with a warm parchment and antique gold palette, editorial left-aligned layouts, and intentional whitespace. It reads as a considered design because it is.
