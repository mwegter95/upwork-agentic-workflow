Architecture

How it works

The AI runs entirely in the visitor's browser. No API calls, no tokens, no monthly bill.

In-Browser RAG Pipeline

User Input
Chat message
Natural language query typed by the customer
→
Retrieval
FAQ vector search
Transformers.js embeddings, cosine similarity, top-3 FAQ chunks selected client-side
→
Generation
In-browser LLM
Qwen2.5-0.5B WASM (Transformers.js) grounded on retrieved context, streams response
→
Response
Cited answer
Answer shown with source citations; order lookup and escalation handled in parallel
Zero recurring cost: The model runs in the browser using WebAssembly. No OpenAI, no Anthropic, no cloud inference bill. The knowledge base is a plain JSON file your team can update. Order status calls your existing API. Zendesk escalation uses the Zendesk Web Widget SDK.

Zendesk Integration Design

How the handoff to Zendesk works

The demo simulates this flow. In production, step 3 is a real Zendesk Web Widget SDK call.

Step 1
Bot detects escalation trigger ("talk to a person," repeated confusion, or explicit request)
Step 2
Check current time in ET. If 9am-6pm, offer live chat. If outside hours, show next window and offer async ticket.
Step 3
Zendesk Web Widget opens pre-filled with conversation transcript and customer context. Or create ticket via REST API with full chat history attached.
Step 4
Ticket ID shown to customer. Bot stays open for follow-up questions while agent reviews the case.
